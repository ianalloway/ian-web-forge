Sports ML: From Predictions to Honest Evaluation

Ian Alloway — ML Engineer · ian@allowayllc.com · ianalloway.xyz
March 2026 · One-page employer case study

Problem

Sports and markets models are easy to oversell with a single headline number (for example “accuracy”). Hiring managers and traders need calibration, time stability, and closing-line-adjacent language—the same bar as serious forecasting and risk workflows.

Approach

I ship a small, production-shaped stack that separates model primitives from evaluation UX:

  • Primitives library (nba-edge on PyPI; source nba-ratings): Elo updates, logistic win probability, implied probability and Kelly helpers—stdlib-first, test-backed, reusable across services.
  • Odds drift watch (odds-drift-watch): monitor sportsbook line movement via FastAPI and webhook alerts for research and decision-support workflows.
  • Evaluation dashboard (nba-clv-dashboard): FastAPI serves metrics JSON; Chart.js surfaces calibration, rolling accuracy, and a CLV-oriented summary block. Swap in real backtest output without rewriting the UI.

Outcome

A coherent narrative for employers: I build correctness-minded ML in a domain where leakage and vanity metrics are common. The dashboard is deliberately an evaluation honesty demo—not a sports integration—mirroring how strong teams review models before production.

Stack

Python · FastAPI · Chart.js · pytest · ruff · GitHub Actions
